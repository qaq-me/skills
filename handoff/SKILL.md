---
name: handoff
description: Compact the current conversation into a handoff document for another agent to pick up.
---

编写一份交接文档，总结当前对话中对下一位 agent 继续工作有用的信息。

交接文档默认保存在当前项目的 `docs/handoffs/` 目录下。如果该目录不存在，先创建它。

生成文件路径时，创建一个形如 `handoff-YYYYMMDD-HHMM-[slug].md` 的 Markdown 文件：

- `YYYYMMDD-HHMM` 使用本地交接时间，精确到分钟。
- `[slug]` 使用英文 kebab-case，由 agent 根据交接重点生成；如果用户传入了参数，优先从参数中提炼。
- 写入前检查目标路径是否已经存在；如果同一分钟同名冲突，追加 `-01`、`-02` 等后缀，直到得到未占用路径。

不要重复当前目录已有内容中已经记录的信息。对于已经存在的需求、计划、决策、问题记录、代码变更或其他交付物，只引用它们的路径或 URL，并简要说明它们包含什么。

如果用户传入了参数，将这些参数视为下一轮会话的重点，并据此调整交接文档的结构和内容。

根据交接重点组织文档结构，优先覆盖：

- 当前目标和用户意图
- 已经达成的结论
- 已完成的工作
- 仍需处理的问题
- 下一步建议
- 相关文件、目录或 URL

保持交接文档简洁、可执行。目标是让新的 agent 能快速恢复上下文，而不是完整复述整个对话。
