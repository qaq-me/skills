# Docs Manager

根据对话和项目变更更新以下项目文档（如果存在的话）：
- 根目录下的 CLAUDE.md、AGENTS.md、README.md。
- 各个层级子目录下的 README.md（不包括第三方仓库、vendor/submodule、下载的外部代码、生成目录、临时目录、归档/备份目录中的 README，除非用户明确要求）。
- docs/ 下的项目文档。

## 更新流程

1. 确认项目根目录和当前 git 状态。
2. 阅读当前项目的规则文档和相关入口文档；如果本轮上下文中已经读取并确认是最新内容，则不必重复阅读。
3. 根据本次变更判断受影响的文档，只更新受项目变更影响的文档，文档更新顺序由你根据变更情况决定。
4. 按项目已有风格、语言、排序规则和目录分层更新文档。
5. 如果发现某个目录可能需要新增 `README.md`，先向用户报告原因和内容概要，得到确认后再创建。
6. 更新完成后做基本一致性检查。

## 验证

根据项目情况执行必要检查，例如：
- 路径引用是否仍然存在。
- 目录说明是否和实际目录一致。
- 多个 agent 入口文档是否需要保持同步。
- Markdown 是否基本可读。
- `git diff --check` 是否通过。
- 是否需要根据项目规则提交或推送。
