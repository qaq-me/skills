# Docs Manager

根据对话和项目变更判断是否需要更新以下文档：

- 根目录下的 `AGENTS.md`、`CLAUDE.md`、`README.md`。
- 子目录下的 `AGENT_GUIDE.md` 和 `README.md`。
- `docs/` 下的项目文档。

不默认更新第三方仓库、vendor/submodule、下载的外部代码、临时目录、归档/备份目录中的文档，除非用户明确要求。

## 更新流程

1. 确认项目根目录和当前 git 状态。
2. 阅读当前项目的规则文档和相关入口文档；如果本轮上下文中已经读取并确认是最新内容，则不必重复阅读。
3. 根据本次变更判断受影响的文档，只更新受项目变更影响的文档。
4. 按项目已有风格、语言、排序规则和目录分层更新文档。
5. 如果发现某个目录需要新增 `AGENT_GUIDE.md` 或 `README.md`，可创建文件并向用户报告。
6. 更新完成后做基本一致性检查。

## 验证

根据项目情况执行必要检查，例如：

- 路径引用是否仍然存在。
- 目录说明是否和实际目录一致。
- README 是否保持人类入口职责。
- `AGENT_GUIDE.md` 是否包含 agent 工作所需的目录上下文。
- 根目录 `AGENTS.md` / `CLAUDE.md` 是否需要同步。
- 多个入口文档之间是否存在过时引用。
- `git diff --check` 是否通过。
- 是否需要根据项目规则提交或推送。

## 文档职责

### 根目录 agent 入口

根目录使用：

- `AGENTS.md`：一般 agent 入口。
- `CLAUDE.md`：Claude Code 入口。

这两个文件用于记录项目目录结构以及 agent guide 索引。两者内容通常需要保持同步。

根目录不使用 `AGENT_GUIDE.md`。

### 子目录 agent 入口

子目录的 agent 专用说明统一使用：

- `AGENT_GUIDE.md`

`AGENT_GUIDE.md` 主要给 agent 读取，通常应包含：

- 当前目录的用途和边界；
- 目录说明；
- 推荐读取顺序；
- 按任务应读取的目录入口

### 人类阅读入口

人类阅读入口使用：

- `README.md`

`README.md` 用于给人快速理解目录内容，通常应包含：

- 这个目录是什么；
- 用户应该先看哪些文件；
- 当前目录下哪些文件最重要；
- 简短的阅读路径或状态说明。

`README.md` 不应承载复杂 agent 执行规则、实验命令契约、生成物维护规则或长篇操作约束。

不是所有目录都必须有 `README.md`。只有确实需要人类直接阅读的目录才新增或维护 README。

## 目录说明排序规则

项目文档中如果包含“目录说明”“文件/目录说明”“项目结构”等入口表，应按以下顺序排列：

1. 文件夹条目在前，普通文件条目在后。
2. 文件夹条目按名称字母序排列。
3. 普通文件条目按名称字母序排列。
4. 排序应尽量与用户在文件管理器或 `ls` 中看到的实际顺序保持一致。
